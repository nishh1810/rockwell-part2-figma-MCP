Working through the Figma-to-code pass went well once I narrowed the scope to layout, spacing, and interaction states. The starter app was intentionally bare, so the fastest path was to build a focused catalog component in `App.tsx` and add a small Zustand store for the filters, selected product, sort option, and clear/remove actions. Using the Figma thumbnail helped establish the overall proportions: compact heading, left filter panel, active filter chips, sort control, and a two-column product grid.

The main friction was limited access to the full Figma file data from this environment. I could not inspect exact tokens directly, so I matched the visible reference as closely as possible from the public preview and iterated on typography, chip styling, and toggle behavior. With more time, I would connect Figma MCP/dev mode directly, extract exact font sizes, colors, and spacing values, then do a final pixel comparison against the frame.
